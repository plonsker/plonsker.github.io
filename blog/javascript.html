Brief Introduction to Ruby Hashes and JavaScript Objects

December 6, 2015

Ruby and JavaScript both have their own types of lists with values we can access. In fact, the syntax of each look nearly identical. I will do my best to explain the nuances here.

Take a look at the following Ruby hash: 

				pets = { 'dog'=> 'Spot', 'cat' => 'Roy', 'fish'=> 'Wanda'}

We have a hash called ‘pets’ that contains the types of pets as keys and the names of each pet as values. We can iterate through the hash, call up some key and value pairs, or simply call individual keys or values. This hash can also be written in the following manner: 

				pets = { dog: 'Spot', cat: 'Roy', fish: 'Wanda'}

Now take a look at the same list as a JavaScript object: 

				var pets = { dog: 'Spot', cat: 'Roy', fish: 'Wanda'}

The whole object is a declared variable in JS. What were keys in the Ruby hash are now JS properties, and what were values are now known as property values. One can access the properties of a JS object by calling them in two ways. If you wished to get the dog, you can either call pets.dog or pets["dog"]. Probably the best way to think about how the two are different from each other is that Ruby considers a hash as simply a list while JS thinks of the object as just one object with many attributes and characteristics. Functionally though, they appear to perform most of the same functions.
